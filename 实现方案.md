## 1. 项目概述

### 1.1 业务背景

在互联网业务场景中，用户 APP 及网页访问会持续产生海量、动态、无序的实时行为日志数据。原始日志数据存在数据冗余、格式杂乱、脏数据较多、存在恶意访问用户等问题，无法直接用于业务统计与运营分析。传统离线计算方式无法满足用户访问流量实时统计、用户行为实时监控、异常访问实时拦截的业务需求。

### 1.2 核心功能

| 序号 | 功能 | 说明 |
|------|------|------|
| 1 | 黑名单恶意用户拦截 | 实时过滤黑名单用户的异常访问数据 |
| 2 | 地区访问流量统计 | 精准统计不同地区的实时访问量 |
| 3 | 1分钟实时在线人次 | 动态监测平台实时访问热度 |
| 4 | 用户行为分布统计 | 梳理浏览、点击、收藏等各类行为占比 |

### 1.3 技术栈

| 层级 | 技术 | 版本 |
|------|------|------|
| 消息队列 | Kafka | 2.6.0 |
| 实时计算 | Spark Streaming (Scala) | 3.4.0 |
| 大数据基础 | Hadoop | 3.3.5 |
| 数据库 | MySQL | 5.7.44（服务器）/ 8.0（开发） |
| 后端 API | SpringBoot 3 (Java, JDK 17) | 3.x |
| 前端 | Vue 3 + ECharts | Vue 3.x |
| Web 服务器 | Nginx | 1.26.1 |
| 构建工具 | SBT (Scala) / Maven (SpringBoot) / Vite (Vue) | — |

## 4. 日志数据格式（六字段制表符分隔）

### 4.1 字段定义

每行日志由 **6 个字段**组成，字段之间以 **Tab 字符（`\t`）** 分隔：

| 序号 | 字段名 | 类型 | 说明 | 示例值 |
|------|--------|------|------|--------|
| 1 | `userId` | Long | 用户唯一标识 | `10001` |
| 2 | `userName` | String | 用户姓名（中文） | `张三` |
| 3 | `action` | String | 用户行为类型（枚举） | `click` |
| 4 | `page` | String | 访问页面 URL | `product-detail.html` |
| 5 | `timestamp` | String | 行为发生时间 | `2025-06-08 10:23:45` |
| 6 | `region` | String | 用户所在地区 | `Guangdong` |

### 4.2 行为类型枚举

| action 值 | 含义 | 说明 |
|-----------|------|------|
| `browse` | 浏览 | 用户浏览页面内容 |
| `click` | 点击 | 用户点击链接/按钮 |
| `favorite` | 收藏 | 用户收藏商品/内容 |
| `purchase` | 购买 | 用户完成购买操作 |
| `exit` | 退出 | 用户离开页面/关闭 |

### 4.6 KafkaProducer 生成逻辑（伪代码）

```scala
object KafkaProducerApp {
  def main(args: Array[String]): Unit = {
    // 1. 配置 Kafka Producer 参数
    val props = new Properties()
    props.put("bootstrap.servers", "localhost:9092")
    props.put("key.serializer", "org.apache.kafka.common.serialization.StringSerializer")
    props.put("value.serializer", "org.apache.kafka.common.serialization.StringSerializer")
    
    val producer = new KafkaProducer[String, String](props)
    val topic = "user_behavior"
    
    // 2. 模拟数据池
    val actions   = Array("browse", "click", "favorite", "purchase", "exit")
    val regions   = Array("Beijing", "Shanghai", "Guangdong", "Shenzhen", "Hangzhou", "Chengdu", "Wuhan", "Nanjing")
    val pages     = Array("index.html", "product-detail.html", "search.html", "checkout.html", "user-center.html")
    val surnames  = Array("张", "李", "王", "赵", "孙", "周", "吴", "郑", "刘", "陈")
    val names     = Array("三", "四", "五", "六", "七", "八", "九", "十", "一", "二")
    
    val random = new Random()
    
    // 3. 无限循环，每 200-500ms 生成一条日志
    while (true) {
      val userId    = 10001 + random.nextInt(100)
      val userName  = surnames(random.nextInt(surnames.length)) + names(random.nextInt(names.length))
      val action    = actions(random.nextInt(actions.length))
      val page      = pages(random.nextInt(pages.length))
      val timestamp = LocalDateTime.now().format(DateTimeFormatter.ofPattern("yyyy-MM-dd HH:mm:ss"))
      val region    = regions(random.nextInt(regions.length))
      
      val logLine = s"$userId\t$userName\t$action\t$page\t$timestamp\t$region"
      
      // 4. 发送到 Kafka
      producer.send(new ProducerRecord[String, String](topic, logLine))
      println(s"[Producer] 发送: $logLine")
      
      Thread.sleep(200 + random.nextInt(300))  // 200~500ms 间隔
    }
  }
}
```

## 5. Kafka 主题设计

### 5.1 主题列表

| 主题名称 | 分区数 | 副本数 | 生产者 | 消费者 | 数据格式 |
|----------|--------|--------|--------|--------|----------|
| `user_behavior` | 3 | 1 | KafkaProducerApp (Scala) | Spark Streaming | Tab 分隔的 6 字段日志 |
| `user_blacklist` | 1 | 1 | BlacklistService (SpringBoot) | Spark Streaming | JSON: `{"type":"add"/"remove", "userId": 10001}` |

### 5.2 创建主题命令

```bash
# 在 Kafka 服务器上执行
kafka-topics.sh --create \
  --topic user_behavior \
  --bootstrap-server localhost:9092 \
  --partitions 3 \
  --replication-factor 1

kafka-topics.sh --create \
  --topic user_blacklist \
  --bootstrap-server localhost:9092 \
  --partitions 1 \
  --replication-factor 1
```

### 5.3 黑名单主题消息格式

```json
// 添加黑名单
{"type": "add", "userId": 10001, "reason": "恶意刷量", "timestamp": "2025-06-08 10:30:00"}

// 移除黑名单
{"type": "remove", "userId": 10001, "timestamp": "2025-06-08 11:00:00"}
```

### 5.4 黑名单实时同步流程

```

                                                           │
                                                           ▼
                                    Spark Streaming 维护内存 Set

```

## 6. Spark Streaming 实时计算逻辑（黑名单 + 四大业务）

### 6.1 整体处理流程

```
Kafka Topic: user_behavior      Kafka Topic: user_blacklist
        │                                │
        ▼                                ▼
  createDirectStream              createDirectStream
        │                                │
        ▼                                ▼
  map: 日志解析                     维护 blacklistSet
  (String → UserBehaviorLog)       (SparkContext 共享变量)
        │                                │
        ▼                                │
  filter: 数据清洗                        │
  · userId 非空                          │
  · action 在枚举范围内                   │
  · 字段数量 = 6                         │
  · 时间戳格式合法                        │
        │                                │
        ▼                                ▼
  ┌─────────────────────────────────────────┐
  │  filter: 黑名单拦截                      │
  │  !blacklistSet.contains(log.userId)     │
  └─────────────────────────────────────────┘
        │
        ├──────────────────────────────────┐
        ▼                                  ▼
  ① 地区统计                           ③ 在线人数
  map → (region, 1)                    map → (userId, 1)
  reduceByKey                          reduceByKeyAndWindow(60s, 10s)
        │                                  │
        ▼                                  ▼
  foreachRDD → MySQL                  foreachRDD → MySQL
  (UPSERT region_stats)               (UPSERT online_count)

        ├──────────────────────────────────┐
        ▼
  ② 行为分布
  map → (action, 1)
  reduceByKey
        │
        ▼
  foreachRDD → MySQL
  (UPSERT behavior_distribution)
```

### 6.2 核心代码框架（Scala）

```scala
import org.apache.spark._
import org.apache.spark.streaming._
import org.apache.spark.streaming.kafka010._
import org.apache.kafka.common.serialization.StringDeserializer

object UserBehaviorStreaming {
  
  def main(args: Array[String]): Unit = {
    // ============ 1. 初始化 SparkConf 和 StreamingContext ============
    val conf = new SparkConf()
      .setAppName("UserBehaviorStreaming")
      .setMaster("local[*]")           // 生产环境改为 yarn
      .set("spark.serializer", "org.apache.spark.serializer.KryoSerializer")
    
    val ssc = new StreamingContext(conf, Seconds(10))  // 微批次间隔 10 秒
    
    // ============ 2. Kafka 消费者配置 ============
    val kafkaParams = Map[String, Object](
      "bootstrap.servers"           -> "localhost:9092",
      "key.deserializer"            -> classOf[StringDeserializer],
      "value.deserializer"          -> classOf[StringDeserializer],
      "group.id"                    -> "user_behavior_group",
      "auto.offset.reset"           -> "latest",
      "enable.auto.commit"          -> (false: java.lang.Boolean)
    )
    
    // ============ 3. 创建 DStream ============
    // 行为日志流
    val behaviorStream = KafkaUtils.createDirectStream[String, String](
      ssc,
      LocationStrategies.PreferConsistent,
      ConsumerStrategies.Subscribe[String, String](Set("user_behavior"), kafkaParams)
    )
    
    // 黑名单指令流
    val blacklistStream = KafkaUtils.createDirectStream[String, String](
      ssc,
      LocationStrategies.PreferConsistent,
      ConsumerStrategies.Subscribe[String, String](Set("user_blacklist"), kafkaParams)
    )
    
    // ============ 4. 黑名单维护（共享变量） ============
    // 使用 Broadcast + Accumulator 模式，或直接从 MySQL 初始化
    // 此处使用内存 Set，由 blacklistStream 持续更新
    var blacklistSet = scala.collection.mutable.Set[Long]()
    
    blacklistStream.foreachRDD { rdd =>
      rdd.collect().foreach { record =>
        val json = parseJson(record.value())
        val userId = json.getLong("userId")
        json.getString("type") match {
          case "add"    => blacklistSet += userId
          case "remove" => blacklistSet -= userId
        }
        println(s"[Blacklist] ${json.getString("type")} user: $userId, current: ${blacklistSet.mkString(",")}")
      }
    }
    
    // ============ 5. 日志解析与清洗 ============
    val cleanedStream = behaviorStream
      .map(_.value())                        // 提取消息体
      .map(parseLogLine)                     // String → Option[UserBehaviorLog]
      .filter(_.isDefined)                   // 过滤解析失败的记录
      .map(_.get)                            // 提取有效记录
      .filter(log => isValidAction(log.action))  // 过滤无效行为类型
      .filter(log => !blacklistSet.contains(log.userId)) // 黑名单拦截
    
    // ============ 6. 四大业务计算 ============
    
    // ----- 6.1 地区访问流量统计 -----
    val regionStats = cleanedStream
      .map(log => (log.region, 1L))
      .reduceByKey(_ + _)
    
    regionStats.foreachRDD { rdd =>
      rdd.foreachPartition { partition =>
        val conn = MySQLConnectionPool.getConnection
        val stmt = conn.prepareStatement(
          """INSERT INTO region_stats (region, visit_count, update_time) 
             VALUES (?, ?, NOW())
             ON DUPLICATE KEY UPDATE visit_count = visit_count + VALUES(visit_count), 
                                     update_time = NOW()"""
        )
        partition.foreach { case (region, count) =>
          stmt.setString(1, region)
          stmt.setLong(2, count)
          stmt.addBatch()
        }
        stmt.executeBatch()
        MySQLConnectionPool.closeConnection(conn)
      }
    }
    
    // ----- 6.2 1分钟实时在线人次（滑动窗口） -----
    val onlineCount = cleanedStream
      .map(log => (1, log.userId))                 // 所有用户映射到 key=1
      .groupByKeyAndWindow(Minutes(1), Seconds(10)) // 窗口 60s, 滑动 10s
      .mapValues(_.toSet.size)                      // 去重计数
      .reduceByKey((a, b) => math.max(a, b))       // 取最大值避免歧义
    
    onlineCount.foreachRDD { rdd =>
      rdd.foreachPartition { partition =>
        val conn = MySQLConnectionPool.getConnection
        val stmt = conn.prepareStatement(
          """INSERT INTO online_count (id, online_users, update_time) 
             VALUES (1, ?, NOW())
             ON DUPLICATE KEY UPDATE online_users = VALUES(online_users), 
                                     update_time = NOW()"""
        )
        partition.foreach { case (_, count) =>
          stmt.setLong(1, count)
          stmt.addBatch()
        }
        stmt.executeBatch()
        MySQLConnectionPool.closeConnection(conn)
      }
    }
    
    // ----- 6.3 用户行为分布统计 -----
    val behaviorDistribution = cleanedStream
      .map(log => (log.action, 1L))
      .reduceByKey(_ + _)
    
    behaviorDistribution.foreachRDD { rdd =>
      rdd.foreachPartition { partition =>
        val conn = MySQLConnectionPool.getConnection
        val stmt = conn.prepareStatement(
          """INSERT INTO behavior_distribution (action_type, count, update_time) 
             VALUES (?, ?, NOW())
             ON DUPLICATE KEY UPDATE count = count + VALUES(count), 
                                     update_time = NOW()"""
        )
        partition.foreach { case (action, count) =>
          stmt.setString(1, action)
          stmt.setLong(2, count)
          stmt.addBatch()
        }
        stmt.executeBatch()
        MySQLConnectionPool.closeConnection(conn)
      }
    }
    
    // ============ 7. 启动流计算 ============
    ssc.start()
    ssc.awaitTermination()
  }
  
  // ============ 辅助方法 ============
  
  /**
   * 解析 Tab 分隔的日志行，返回 Option[UserBehaviorLog]
   * 各字段按 Tab(\t) 分隔: userId\tuserName\taction\tpage\ttimestamp\tregion
   */
  def parseLogLine(line: String): Option[UserBehaviorLog] = {
    try {
      val parts = line.split("\t")
      if (parts.length != 6) return None    // 字段数量不符合
      
      val userId   = parts(0).toLong
      val userName = parts(1).trim
      val action   = parts(2).trim
      val page     = parts(3).trim
      val timestamp = parts(4).trim
      val region   = parts(5).trim
      
      // 校验必填字段非空
      if (userName.isEmpty || action.isEmpty || region.isEmpty) return None
      
      Some(UserBehaviorLog(userId, userName, action, page, timestamp, region))
    } catch {
      case _: Exception => None   // 解析异常，返回 None
    }
  }
  
  /**
   * 校验行为类型是否合法
   */
  def isValidAction(action: String): Boolean = {
    val validActions = Set("browse", "click", "favorite", "purchase", "exit")
    validActions.contains(action)
  }
}

/**
 * 用户行为日志数据模型
 */
case class UserBehaviorLog(
  userId:    Long,    // 用户 ID
  userName:  String,  // 用户姓名
  action:    String,  // 行为类型
  page:      String,  // 访问页面
  timestamp: String,  // 行为时间
  region:    String   // 所在地区
)
```

### 6.3 四大计算详解

| 计算任务 | 算子 | 窗口 | 输出目标 | 更新策略 |
|----------|------|------|----------|----------|
| 地区流量统计 | `map` + `reduceByKey` | 无（累加） | `region_stats` 表 | 每次累加到已有值 |
| 1 分钟在线人次 | `groupByKeyAndWindow` | 窗口 60s, 滑动 10s | `online_count` 表 | 替换为当前窗口值 |
| 行为分布统计 | `map` + `reduceByKey` | 无（累加） | `behavior_distribution` 表 | 每次累加到已有值 |

### 6.4 数据清洗规则汇总

| 问题类型 | 检测规则 | 处理方式 |
|----------|----------|----------|
| 空值数据 | `userId` 为空字符串、`action` 为空、`region` 为空 | 丢弃 |
| 字段缺失 | `split("\t").length != 6` | 丢弃 |
| 行为类型异常 | `action` 不在 `[browse, click, favorite, purchase, exit]` 中 | 丢弃 |
| 格式异常 | `userId` 不是有效数字、时间戳格式错误 | 丢弃 |
| 恶意用户 | `userId` 在黑名单集合中 | 拦截（丢弃） |

```sql
-- 表1：地区访问量统计
CREATE TABLE region_stats (
    region      VARCHAR(50)  NOT NULL PRIMARY KEY COMMENT '地区名称',
    visit_count BIGINT       NOT NULL DEFAULT 0 COMMENT '累计访问次数',
    update_time DATETIME     DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '最后更新时间'
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='地区实时访问量统计';

-- 表2：实时在线人次（单行表，始终 id=1）
CREATE TABLE online_count (
    id           INT          NOT NULL PRIMARY KEY DEFAULT 1 COMMENT '固定主键',
    online_users BIGINT       NOT NULL DEFAULT 0 COMMENT '1分钟内在线人次',
    update_time  DATETIME     DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '最后更新时间'
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='1分钟实时在线人次';

-- 表3：用户行为分布统计
CREATE TABLE behavior_distribution (
    action_type VARCHAR(50)  NOT NULL PRIMARY KEY COMMENT '行为类型(browse/click/favorite/purchase/exit)',
    count       BIGINT       NOT NULL DEFAULT 0 COMMENT '累计次数',
    update_time DATETIME     DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '最后更新时间'
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='用户行为类型分布统计';

-- 表4：黑名单管理表（由 SpringBoot 维护）
CREATE TABLE blacklist (
    user_id      BIGINT       NOT NULL PRIMARY KEY COMMENT '被拉黑的用户ID',
    user_name    VARCHAR(50)  DEFAULT '' COMMENT '用户姓名',
    reason       VARCHAR(255) DEFAULT '' COMMENT '拉黑原因',
    create_time  DATETIME     DEFAULT CURRENT_TIMESTAMP COMMENT '添加时间'
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='黑名单用户表';

-- 初始化行为分布表（5种行为类型）
```

```javascript
  const { data } = await fetchOverview()
```

```nginx
        proxy_set_header X-Real-IP $remote_addr;
    }
}
```

## 附录：关键注意事项

1. **JDK 版本兼容**：Spark 3.4.0 在 JDK 8 上运行；SpringBoot 3 需要 JDK 17。两者互不冲突，Spark 用 SBT + JDK 8，SpringBoot 用 Maven + JDK 17。
2. **MySQL 5.7 vs 8.0**：SQL 语法兼容，注意 `serverTimezone=Asia/Shanghai` 参数在 5.7 中不需要。
3. **Kafka offset 管理**：使用 `enable.auto.commit=false` 配合 Spark Streaming 的 checkpoint 机制，避免数据丢失或重复消费。
4. **黑名单同步延迟**：黑名单指令从 SpringBoot 写入 Kafka 再到 Spark Streaming 消费更新内存 Set，有约 10 秒延迟（取决于批次间隔），属于正常范围。
5. **Spark checkpoint**：建议开启 `ssc.checkpoint("hdfs://...")` 以支持故障恢复，特别是在 `groupByKeyAndWindow` 场景下是必需的。
