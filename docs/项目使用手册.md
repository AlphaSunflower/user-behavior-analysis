## 二、首次使用

### 1. 克隆项目

```bash
git clone https://github.com/AlphaSunflower/user-behavior-analysis.git
cd user-behavior-analysis
```

### 2. 初始化 MySQL

```bash
mysql -u root -p123456 < docs/sql/schema.sql
```

### 3. 编译各模块

```bash
# kafka-producer
cd kafka-producer && sbt compile && cd ..

# kafka-spark
cd kafka-spark && sbt compile && cd ..

# backend
cd backend && mvn compile && cd ..

# frontend
cd frontend && npm install && cd ..
```

### 4. 确保服务器服务已启动

Kafka（192.168.100.140:9092）必须运行。详见 [服务器启动指南](./服务器启动指南.md)。

---

## 三、启动系统

### 方式一：一键启动（推荐）

双击 `docs/start-all.bat`，自动按顺序启动 4 个服务。

### 方式二：手动逐个启动

打开 **4 个终端**，按以下顺序执行：

| 顺序 | 终端 | 命令 | 说明 |
|------|------|------|------|
| 1 | PowerShell | `docs\scripts\start-streaming.bat` | Spark Streaming（必须先启动） |
| 2 | PowerShell | `docs\scripts\start-producer.bat` | Kafka 生产者（等 Streaming 就绪后） |
| 3 | PowerShell | `docs\scripts\start-backend.bat` | SpringBoot API（端口 8080） |
| 4 | PowerShell | `docs\scripts\start-frontend.bat` | Vue 前端（端口 5173） |

### 方式三：命令行

```bash
# 终端 1
cd kafka-spark && sbt "runMain streaming.UserBehaviorStreaming"

# 终端 2
cd kafka-producer && sbt "runMain producer.KafkaProducerApp"

# 终端 3
cd backend && mvn spring-boot:run

# 终端 4
cd frontend && npm run dev
```

## 五、功能说明

### 5.1 仪表盘（Dashboard）

打开 http://localhost:5173 后自动显示：

| 区域 | 内容 | 刷新频率 |
|------|------|----------|
| 顶部 | 实时在线人次 + 累计访问量 | 5 秒 |
| 中左 | 地区访问流量（柱状图） | 5 秒 |
| 中右 | 用户行为分布（饼图） | 5 秒 |
| 下左 | 设备类型分布（柱状图） | 5 秒 |
| 下右 | 来源渠道分布（饼图） | 5 秒 |
| 底部 | 黑名单管理面板 | 手动 |

### 5.2 黑名单管理

- **添加**：点击 `+ 添加` → 输入用户ID、用户名、原因 → `确认添加`
- **移除**：点击列表中的 `移除` 按钮
- 添加/移除通过 Kafka 实时同步到 Spark Streaming，约 10 秒内生效

### 5.3 API 接口

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/api/stats/overview` | 全部统计数据 |
| GET | `/api/stats/regions` | 地区流量 |
| GET | `/api/stats/online` | 实时在线 |
| GET | `/api/stats/behaviors` | 行为分布 |
| GET | `/api/blacklist` | 黑名单列表 |
| POST | `/api/blacklist` | 添加黑名单 |
| DELETE | `/api/blacklist/{userId}` | 移除黑名单 |

## 七、常见问题

### Q1: Producer 报 "metadata after timeout"

确认 Kafka 服务器正在运行，且 `kafka-producer/build.sbt` 中 kafka-clients 版本为 3.3.2。

### Q2: 中文乱码

在 PowerShell 中先执行 `chcp 65001`，再运行 sbt 命令。启动脚本已自动设置。

### Q3: SBT 锁冲突

Producer 和 Streaming 已拆分为独立项目（不同目录），同时运行不会冲突。

### Q4: Spark Streaming 启动后没数据

1. 确保 Producer 已启动并在发送数据
2. 检查 Kafka Topic 是否存在：`/home/hadoop/opt/app/kafka/bin/kafka-topics.sh --list --bootstrap-server 192.168.100.140:9092`

### Q5: 前端页面空白

1. 确保 SpringBoot 已启动（`curl http://localhost:8080/api/stats/overview`）
2. 检查浏览器控制台是否有 CORS 错误

## 八、部署到生产服务器

```bash
# 1. 打包 kafka-spark
cd kafka-spark && sbt assembly
# 产物：target/scala-2.12/UserBehaviorAnalysis-assembly-1.0.jar

# 2. 打包 backend
cd backend && mvn package -DskipTests
# 产物：target/user-behavior-backend-1.0.jar

# 3. 打包 frontend
cd frontend && npm run build
# 产物：dist/

# 4. 上传并部署（参见服务器启动指南）
```

### 部署前注意

将 `build.sbt` 中 spark-streaming/spark-sql 的 `% "compile"` 改回 `% "provided"`，避免 JAR 体积过大。
